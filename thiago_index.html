Swift 2.0 - Storage Policies

Ability to choose how and where to store objects

  • How: 2x or 3x replication, EC
  • Where: HDs, SSDs, Kinetic drives or

    • a Scale-out File System

Swift-on-File

A Swift Object Server implementation which allows objects created using the Swift API to be accessed as files on a POSIX filesystem

  • Use a scale-out filesystem to store objects (e.g., GlusterFS, GPFS)
  • Map URL to file path
